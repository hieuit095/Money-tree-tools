# 24/7/365 Operation

This project is designed to run continuously on low-power devices (SBCs, mini PCs, and small servers). The deployment is optimized for stability, bounded resource usage, and predictable recovery behavior.

## Stability

- The dashboard runs as a systemd service (`income-manager.service`) with automatic restart.
- The dashboard includes an internal watchdog loop to monitor and recover managed services.
- Docker-managed services are configured with restart policies to recover after crashes and reboots.

## Performance and Resource Control

- Docker services use CPU/RAM/pid limits to prevent one container from destabilizing the host.
- Docker logs are capped/rotated to reduce disk wear and prevent disk-fill outages.
- The host is optimized with ZRAM swap and VM tuning for small-memory devices.
- Optional maintenance can prune unused Docker images/build cache to keep disk usage stable over time.

## ARM / SBC Compatibility

- ARM deployments can enable multi-arch support (binfmt/qemu) for amd64-only images when necessary.
- Prefer native images by setting `TARGET_PLATFORM` (and per-service `*_PLATFORM` overrides) to match your board.
- To validate common ARM/SBC prerequisites:

```bash
python scripts/check_arm_requirements.py
```

## Operational Recommendations

- Use a reliable power supply/UPS where possible and ensure good cooling (passive devices will throttle under load).
- Keep the filesystem healthy and leave free disk space for Docker pulls and log rotation.
- For constrained devices, prefer native images and conservative CPU/RAM limits.

## Per-Device Profiles

When deploying multiple devices, you can define profiles in `inventory.yaml` and reference them from each device. Profile values are emitted as environment variables (uppercased) into the encrypted device config.

Example:

```yaml
profiles:
  sbc_low:
    ZRAM_SIZE_MB: 1024
    WATCHDOG_INTERVAL_SEC: 300

devices:
  - ip: "192.168.1.15"
    user: "orangepi"
    pass: "orangepi"
    box_id: "op-192-168-1-15"
    profile: "sbc_low"
    env:
      TARGET_PLATFORM: "linux/arm64"
```
