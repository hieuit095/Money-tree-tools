## Actions Taken

### Removal (Non-Allowed Services)

- Removed all codebase references to EarnApp from:
  - backend service maps and routes
  - watchdog recovery
  - dashboard service selector
  - internal docs
- Deleted service-specific helper scripts that contained plaintext credentials and were dedicated to removed services.
- Scrubbed plaintext credentials from tracked inventory files.
### Addition (Missing Required Services)

- Added Docker-managed services:
  - Grass
  - Mysterium
  - Pawns
  - PacketShare
  - ProxyRack
  - Bitping
- Added host-native managed service:
  - Wipter (managed as `wipter.service` via systemd)
  - Uprock (managed as `uprock.service` via systemd)

### Supply Chain Controls

- Pinned all Docker images by immutable sha256 digest in `docker-compose.yml`.
- Recorded the pinned image inventory in `reports/image-digests.json`.
### Hardening

- Enforced Docker log rotation (5MB max, 1 file) for all services.
- Enforced container CPU/memory/PID limits in `docker-compose.yml`.
- Added Mysterium TequilAPI healthcheck in the watchdog (restarts container on failed `/healthcheck`).
- Implemented encrypted-at-rest configuration storage (`.env.enc`) and removed reliance on persistent plaintext `.env`.
